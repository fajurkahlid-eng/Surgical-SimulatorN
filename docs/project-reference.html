المرجع الشامل للمشروع (Surgical Training Environment VR)

وثيقة تفصيلية لكل جدول، حقل، مسار API، مسار واجهة، ملف محاكي، مفتاح تخزين، ووظيفة — مع سبب الوجود والمهمة. مبنية على الكود الحالي (JWT + bcrypt).

1 — هيكل المشروع (المجلدات الرئيسية)

المسارالغرض

2 — قاعدة البيانات MySQL — الجداول والحقول حرفياً

المصدر: server/schema.sql. اسم القاعدة الافتراضي: surgical_training.

2.5 الفهارس (Indexes)

  • idx_sessions_trainee على SESSIONS(TraineeID) — تسريع استعلامات المتدرب
  • idx_sessions_date على SESSIONS(Date) — ترتيب زمني
  • idx_reports_session على REPORTS(SessionID)
  • idx_trainees_email على TRAINEES(Email) — بحث تسجيل الدخول

2.6 البذور (INSERT الأولي في schema.sql)

  • دورة CourseID = 1: اسم عربي، نوع VR، مدة 15.
  • حساب مدرّب افتراضي: البريد instructor@example.com، كلمة المرور في الـ seed نصّ (يُفضّل تغييرها؛ أول تسجيل دخول قد يحوّلها لـ bcrypt).

4 — مصادقة الخادم (server/auth.js)

دالة / وسيطالمهمة

needsRehash(stored)هل يجب استبدال التخزين بـ bcrypt

8.3 sessionBridge.js — الحقول في الحمولة

getSessionContext() يُرجع: sessionId, traineeId, courseId, traineeName, difficulty, autoStart, nameLocked, exitToApp.

buildLastSessionPayload يبني: sessionId, traineeId, totalScore, accuracyScore, speedScore, stepsCompleted, totalSteps, durationSeconds, comments (JSON مختصر).

persistProcedureResult يكتب lastSessionResult إما مع sessionId أو evaluationOnly: true إن لا توجد جلسة DB.

10 — تدفقات مختصرة

  1. تسجيل دخول: POST login → token + user → localStorage → طلبات لاحقة بـ Bearer.
  2. تدريب: POST sessions → فتح محاكي بمعاملات → إنهاء إجراء → lastSessionResult → reports → PATCH end + POST report إن وُجد sessionId.
  3. مدرّب: JWT بدور instructor → GET admin/*.
